## Medium Priority

### 1. K-lagged CN is formally O(dt) globally for nonlinear diffusion

**File:** `src/scm/time_stepping/implicit.py`

**This is the standard "semi-implicit" formulation** used in virtually all NWP models
(WRF, ECMWF IFS, PALM, etc.) — lagging K at `tⁿ` on both sides of the CN equation is the
intended design, not a bug.

For a *linear* problem (K constant), using the same K on both sides is exact and the scheme
is fully second-order. For a *nonlinear* problem (K = K(φ)), the lagged K introduces an
extra O(dt²) local truncation error proportional to `∂K/∂φ · dφ/dt`, which accumulates to
O(dt) globally. For the slowly-varying eddy diffusivities typical of BL simulations at
dt = 5–60 s this prefactor is small and the practical impact is negligible.

The AB2 extrapolation of explicit sources S also has O(dt²) local error — the same formal
order — so there is no practical inconsistency between the two parts of the scheme.

**If higher accuracy in K is ever needed:** a single predictor-corrector iteration (predict
`y*` with `Kⁿ`, re-diagnose `K*` from `y*`, re-solve with `K_mid = (Kⁿ + K*)/2`) gives a
second-order approximation of `K(t + dt/2)` at the cost of one extra closure evaluation per
step. This is non-standard and likely unnecessary for typical use.

### 2. Semi-implicit TKE shear production

**File:** `src/scm/time_stepping/implicit.py`, `src/scm/mynn/model.py:92`

In implicit mode `qke_tends = diag.qke_P_S + diag.qke_P_B` is fully explicit and enters
the AB2 extrapolation.  `qke_P_S = K_m * |du/dz|²` is the term most responsible for the AB2
computational-mode build-up in stable boundary layers (see `workspaces/benchmark/TIMESTEP_GUIDE.md`).

**Fix:** quasi-linearise P_S around the current state — analogous to how dissipation is
already handled — and add it as a negative entry in the CN `decay` diagonal:

```python
# in _apply_cn (implicit.py), alongside qke_decay:
qke_growth = diag.qke_P_S / jnp.clip(y.qke, min=consts.qke_min)  # [1/s], always >= 0
net_decay = qke_decay - qke_growth  # negative when production dominates
decay = jnp.zeros_like(phi).at[4].set(net_decay)
```

Caveat: the CN solver assumes the diagonal is positive (unconditional positivity). When
`net_decay < 0` (convective cases where production >> dissipation) additional care is needed —
either clamp to zero or accept explicit treatment there. For stable BL (dissipation dominates)
this is always safe.

## Low Priority

### 3. Replace AB2 with IMEX-RK2 to eliminate the computational mode

**File:** `src/scm/time_stepping/explicit.py`, `src/scm/time_stepping/implicit.py`

AB2 is a **multistep** method — it carries two characteristic roots. The physical root
tracks the true solution; the parasitic (computational-mode) root oscillates at period
`2×dt` and is never fully damped when the implicit diffusion coefficient K is small (e.g.
post-transition stable BL). This is the root cause of the dt=1 s error anomaly documented
in `TIMESTEP_GUIDE.md`.

An **IMEX-RK** scheme replaces AB2 with a *one-step* Runge-Kutta method that has only one
characteristic root — the computational mode does not exist. The simplest second-order
variant (CN for diffusion + explicit midpoint/Heun for sources) costs 2 model evaluations
per step (vs 1 for AB2) but offers:

- No parasitic oscillation in any K regime.
- No warmup step (one-step method is self-starting).
- A wider stable dt range for explicit sources (Runge-Kutta stability regions are larger
  than AB2's for most problems).

Sketch of the CN+IMEX-RK2 step:

```
Stage 1: S¹ = model(tⁿ, yⁿ)
         y* = CN_solve(yⁿ, S¹, dt/2)          # half-step implicit diffusion

Stage 2: S² = model(tⁿ + dt/2, y*)
         y^(n+1) = CN_solve(yⁿ, S², dt)        # full-step CN with midpoint source
```

The extra model evaluation is the main cost. Since `model()` is JIT-compiled, the
wall-clock overhead is roughly 2× per step; whether this is offset by using larger dt
depends on the case.

**When to consider:** if the AB2 computational mode causes persistent problems in strongly
stable or very short-dt runs, and the 2× eval cost is acceptable.

### 4. Geostrophic advection gradient has a top-boundary error

**File:** `src/scm/mynn/model.py:100-104`

`dug_dz` is computed on half levels via `d_dz(..., top=0.0)` then averaged back to full
levels via `(dug_dz[1:] + dug_dz[:-1]) / 2`. For **interior** levels the round-trip produces
the standard centred difference `(u_geo[j+1] - u_geo[j-1]) / (2*dz)` — correct. At the
**top** level (`j = Nz-1`) the averaging mixes the last interior difference with the zero top
boundary condition, halving the result:

```
dug_dz_full[Nz-1] = (u_geo[Nz-1] - u_geo[Nz-2]) / (2*dz)   # should be / dz
```

For smoothly-varying geostrophic profiles this is minor (the top level is usually above the BL),
but it causes a systematic under-estimate of geostrophic temperature advection there.

**Fix:** compute directly on full levels with a one-sided difference at both boundaries:

```python
dug_dz = jnp.concatenate([
    jnp.array([(u_geo[1] - u_geo[0]) / dz]),
    (u_geo[2:] - u_geo[:-2]) / (2 * dz),
    jnp.array([(u_geo[-1] - u_geo[-2]) / dz]),
])
```
